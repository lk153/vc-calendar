# vc-calendar

Internal room booking app — Next.js 15 / Postgres / Prisma / Auth.js.

## Setup

```bash
cp .env.example .env       # fill DATABASE_URL, AUTH_SECRET, Google + Resend
pnpm install
pnpm prisma migrate dev --name init   # generates baseline schema migration
pnpm prisma migrate deploy            # applies booking_no_overlap exclusion-constraint migration
pnpm db:seed                          # admin/admin123! + user/user123! + sample rooms
pnpm dev
```

Note: the `20260429000001_booking_no_overlap` migration adds the `btree_gist` extension and a GiST exclusion constraint that makes location double-booking impossible at the DB layer (catches `23P01` in `lib/bookings/service.ts`). Order: init (autogenerated from schema) runs first, exclusion constraint after.

## Stack

- Next.js 15 App Router (route groups: `(user)`, `(admin)`)
- Auth.js v5 — Credentials (bcrypt) + Google. Invite-only (no public register).
- Postgres + Prisma. Exclusion constraint for room conflict.
- Tailwind w/ design tokens from `design/DESIGN.md` (Lumina Executive — Emerald + Manrope/Inter).
- Resend for create/update/cancel emails (no-op without `RESEND_API_KEY`).

## Conflict rules
- Location: DB exclusion constraint (hard).
- Organizer: serial check, hard block.
- Attendees: warn-and-allow override.
